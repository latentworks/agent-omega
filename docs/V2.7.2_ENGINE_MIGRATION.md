# Agent Omega v2.7.2 engine migration

Agent Omega v2.7.2 requires the matching `omega-v2.7.2` engine release. It
adds an engine-owned terminal checkpoint: a prose plan is read only after it
is durable, while a receipt-backed final candidate is held out of the
transcript until the same isolated review and lifecycle gates choose what is
safe to expose. Neither path depends on a smaller model choosing the control
tool.

## Windows upgrade behavior

On startup, a recognized Agent Omega installation refreshes only the managed
`task-quality` plugin folder and reconciles its two required plugin entries.
It preserves personal model/provider settings, council data, memory, and
other user plugins. A foreign OpenCode configuration is left untouched and
Agent Omega refuses to start task work rather than modifying it.

## Upgrade

1. Update Agent Omega to v2.7.2.
2. Download the matching native asset and checksum file from the
   [omega-v2.7.2 engine release](https://github.com/latentworks/opencode-omega/releases/tag/omega-v2.7.2).
3. Verify the SHA-256, replace the local engine binary, and restart Agent
   Omega. Windows will refresh the managed safety plugin on that first start.

Do not mix this app with an upstream OpenCode binary or an earlier Omega
engine. A health-check failure is a version mismatch, not a prompt setting to
work around.
