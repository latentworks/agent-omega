# Agent Omega v2.7.1 engine migration

Agent Omega v2.7.1 requires the matching `omega-v2.7.1` engine release. This
patch fixes a lifecycle deadlock in internal verification continuations: they
are now durably marked as engine automation rather than new user requests.

The app will refuse an older engine at startup. That is intentional: the
required health contract now includes `internal-automation` in addition to the
protocol-2 lifecycle, review, and tool-admission features.

## Upgrade

1. Update Agent Omega to v2.7.1 and run `node setup.mjs` from the app source.
   Existing personal model, provider, memory, and council configuration is
   preserved.
2. Download the native asset and checksum file from the
   [omega-v2.7.1 engine release](https://github.com/latentworks/opencode-omega/releases/tag/omega-v2.7.1).
3. Verify the asset SHA-256, replace the local engine binary, then start a new
   Agent Omega session.

Do not mix this app with an upstream OpenCode binary or an earlier Omega
engine. A health-check failure is a version mismatch, not a prompt setting to
work around.
