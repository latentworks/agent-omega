# v2.6 engine migration

Agent Omega v2.6.1 requires its matching forked engine. The task-quality
lifecycle depends on engine protocol/capability enforcement; an upstream
OpenCode binary or an earlier Agent Omega engine is intentionally rejected at
startup instead of running the feature as an unenforced prompt convention.

## What the matching engine enforces

The v2.6 engine carries the authority for the lifecycle; the configuration
plugins cannot substitute for it. In particular, it:

- accepts one structured `submit_review` result as the reviewer's terminal
  submission, rather than treating free-form reviewer text as an approval;
- rejects an internally contradictory `pass` verdict that also contains
  findings, so a reviewer cannot describe a failed criterion and authorize it;
- requires an artifact reviewer to inspect the engine-supplied workspace with a
  read-only tool before it may submit a verdict;
- records the final task agent after tool-argument transforms and the
  engine-attested count of completed built-in child reads in bounded execution
  provenance;
- binds direct task children to the engine-issued parent grant, rather than to
  caller-supplied session metadata;
- persists the exact reviewed plan generation and an external-user approval
  before mutation, and keeps artifact review closed while execution is pending;
- separates plan text, artifact text, and sanitized execution receipts from the
  builder conversation, hidden reasoning, tool arguments, and raw tool output;
  any workspace inspection occurs only through engine-owned read-only tools.
- supplies artifact reviewers with engine-attested checkpoint, approval,
  direct-task, child-read, receipt, and pending-execution facts while clearly
  separating them from optional command-output receipts.

The bundled skill router also follows the active local model for each turn.
Cloud-led turns require an explicit local classifier override; a failed
classifier fails its routing ticket, enters a short cooldown, and later reports
recovery. Internal subagent messages are not routing input.

## Windows upgrade

1. Update the repository to v2.6.1 and run `node setup.mjs`. Existing model,
   council, and memory data are preserved.
2. Download `opencode.exe` and `SHA256SUMS-v2.6.1.txt` from the
   [v2.6.1 release](https://github.com/latentworks/agent-omega/releases/tag/v2.6.1).
3. Put `opencode.exe` at `engine\opencode.exe` in the repository root.
4. Verify its SHA-256 before running it:

   ```powershell
   Get-FileHash .\engine\opencode.exe -Algorithm SHA256
   ```

   Expected value:

   ```text
   4a1c01e4c3192fb7aaac924e1aeccf9190936ad35a092e22fc9260b618eb3799
   ```

5. Rebuild with `dotnet build -c Release` and launch the rebuilt application.

`node scripts\smoke.mjs` checks that the engine file is present, but only the
app startup handshake proves v2.6 engine compatibility.

## Platform scope

The v2.6.1 engine asset is Windows-only. A matching macOS v2.6 engine asset is
not published in this release, so macOS users should stay on the last release
with a matching macOS engine. Do not pair v2.6 source with an older macOS engine.
