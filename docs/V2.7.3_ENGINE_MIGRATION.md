# Agent Omega v2.7.3 engine migration

Agent Omega v2.7.3 must be paired with engine tag `omega-v2.7.3`. The public
task-quality protocol remains protocol 2 with the same ten advertised
capabilities; this patch tightens internal settlement and recovery behavior
without changing provider configuration, stored conversations, or plugin data.

## What changed

- The sidecar now emits an identity-bearing `turn-settled` event only after the
  exact ACP prompt or command has physically finished. Session replacement can
  no longer advance on an earlier UI-only `turn-end`.
- An artifact CRAP report that returns as prose without a new post-review
  receipt gets one bounded engine-attested continuation. The lifecycle stays
  pending until fresh evidence exists and fails closed if that continuation
  cannot run.
- Local Qwen plain-text CRAP reviews run with hidden thinking disabled. The
  override is restricted to literal loopback/private inference endpoints;
  remote OpenAI-compatible Qwen providers are untouched.
- The measured local `qwen3-coder-80b` CRAP window is 300 seconds. Other local
  and cloud review budgets keep their existing bounds.

## Upgrade

1. Pull Agent Omega v2.7.3.
2. Download the engine asset for your platform from
   [`omega-v2.7.3`](https://github.com/latentworks/opencode-omega/releases/tag/omega-v2.7.3).
3. Verify the asset against `SHA256SUMS-omega-v2.7.3.txt` and
   `SOURCE-PROVENANCE.json`.
4. Replace the previous engine binary and rebuild/relaunch the desktop app.

Engine-fork adopters should build from the annotated `omega-v2.7.3` tag with
`OPENCODE_VERSION=2.7.3-omega.1` and `OPENCODE_CHANNEL=omega`. Upstream OpenCode
and earlier Agent Omega engines remain intentionally incompatible.

## AMD Vulkan/RADV serving note

For the affected Qwen3-Coder-Next 80B llama.cpp path, remove the five
speculative-decoding flags listed in [Model Tunes](../TUNES.md#qwen3-coder-next-80b-on-amd-vulkanradv).
Keep `-ngl 999`; it is main-model offload, not the `-ngld 999` draft-model
flag. Three identical frozen lifecycles completed after the speculative group
was removed, making that group the suspect on the tested path—not a universal
AMD diagnosis.
