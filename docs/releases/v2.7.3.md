# v2.7.3 — exact settlement and robust single-model recovery

## What changed

- The desktop sidecar now distinguishes UI completion (`turn-end`) from exact
  physical provider/engine settlement (`turn-settled`). Every tracked prompt
  and command carries one stable identity from start through settlement;
  queued session changes wait for the final boundary.
- Same-model CRAP artifact recovery is now bounded and fail-closed. A prose
  review that lacks a new receipt queues at most one engine-attested corrective
  continuation and cannot silently authorize completion.
- The engine gives the measured local Qwen3-Coder-80B CRAP path a five-minute
  review budget and disables hidden reasoning only for local OpenAI-compatible
  Qwen plain-text review calls. Cloud/custom Qwen endpoints keep their provider
  behavior.
- The live lifecycle harness and its transport broker are now public and
  configurable for Windows test hosts with SSH telemetry access to the model
  server. They retain full provider request/SSE evidence, enforce token liveness
  through keepalives, bound zombie drains, validate metrics/timings, and wait
  for the exact settlement identity.
- The AMD Vulkan/RADV serving note records the five speculative-decoding flags
  removed from the tested Qwen3-Coder-Next 80B command. The identical frozen
  32K/4K thinking-off lifecycle then completed three consecutive times with
  zero pending execution and no stream stall.
