# Agent Omega v2.6.1 — lifecycle hardening release record

## What changed

- Added the task-quality lifecycle: qualifying work is planned, independently
  reviewed, repaired, explicitly approved by the user, verified, and reviewed
  again before completion.
- Added a clean-room adversarial fallback when no independent reviewer is
  available.
- Added startup capability enforcement so an older or upstream engine cannot
  silently run the lifecycle unenforced.
- Added engine-enforced reviewer completion: a reviewer submits one structured
  verdict, and completed-artifact review requires read-only inspection of the
  engine-supplied workspace.
- Added durable execution provenance: task receipts record the final
  transform-resolved task agent and engine-attested completed child-read count,
  while keeping builder context, tool arguments, and raw output outside the
  review envelope; workspace claims are checked through engine-owned read-only
  tools.
- Added exact-generation approval and completion gates: mutation requires a
  durable external-user approval, and artifact review remains closed until
  execution receipts exist with no pending execution.
- Hardened substantive scope changes against concurrent receipt settlements:
  stale approval is revoked across the full bounded pending-execution window,
  including chained compare-and-swap conflicts.
- Hardened routing: the classifier follows the active local model, cloud leads
  require an explicit local override, classifier failures use a bounded
  cooldown, and internal subagent messages are excluded from routing.
- Added the persistent bottom operator runner showing the active model and the
  available token, context-limit, and provider-quota data.

## Architecture checks covered by the repository tests

The focused logic suites cover the lifecycle and router contracts rather than
only prompt wording. They exercise structured reviewer submission, the
completed-artifact workspace-evidence requirement, terminal reviewer
submission, final task-agent provenance after transforms, attested child-read
provenance, exact external-user approval, pending-execution blocking, and the
plan/artifact/receipt text boundaries. Router coverage includes active-model
affinity, explicit cloud fallback, timeout or unavailable classifier handling,
cooldown recovery, and internal-subagent filtering.

## Scope not verified for this release

- A matching macOS v2.6 engine package and macOS runtime path were not verified;
  this release therefore ships a Windows engine asset only.
- The engine's full repository suite did not complete inside a ten-minute local
  run. The changed SDK/API suite, typecheck, compiled smoke test, and Agent Omega
  suite passed; this timeout is retained as a verification gap.
- The optional paid cloud/API browser end-to-end suite was not run. It is not a
  CI or release gate because it incurs provider charges.
- A positive terminal artifact-review result was not re-established on the
  final packaged binary after the last evidence-packet hardening. Final-binary
  execution and negative/fail-closed paths were verified; the local reviewer
  remained nondeterministic at structured final submission.
