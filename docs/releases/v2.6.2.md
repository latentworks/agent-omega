# Agent Omega v2.6.2 — public engine source release record

## What changed

- Published the source-complete Agent Omega engine fork under the Latentworks
  account with its upstream OpenCode history and MIT license intact.
- Bound the app release to engine tag `omega-v2.6.2`, source commit
  `f32f5c9c5e143651594e901f0501eac5acde1a6d`, upstream base
  `5c23e88419c4743b9be42cea132f2fb1e6cb63ff`, and build ID
  `f32f5c9c5e14-aea92940cb6a`.
- Pinned the engine's Bun toolchain, lockfile, Git dependency, and model-catalog
  snapshot so a release build does not silently consume moving network state.
- Published native engine assets for Windows x64, Linux x64/arm64, macOS Intel,
  and macOS Apple Silicon with checksums and machine-readable provenance.
- Replaced the obsolete claim that the engine source was private or that v2.6
  was Windows-only. The desktop app remains Windows x64 and macOS Apple
  Silicon; Linux and macOS Intel are engine-only release targets.

## Verified before publication

| Check | Result |
| --- | --- |
| Changed engine packages | Core, engine, and plugin typechecks passed |
| Focused lifecycle/API tests | 67 passed, 0 failed |
| Clean local Windows proof | Engine launched; health exposed protocol 1, all required features, and the expected source identity; two same-runner builds were byte-identical |
| Native release matrix | Windows x64, Linux x64/arm64, macOS Intel, and macOS Apple Silicon each built twice, compared byte-for-byte, launched natively, and passed health/source-identity verification |
| Public source | Fork `main` and annotated tag `omega-v2.6.2` resolve to the source commit above; default branch is `main` |
| Privacy | Exact source diff, commit metadata, release notes, manifests, and asset names passed the outbound privacy scan; private-network values retained in tests are synthetic RFC1918 boundary fixtures |
| Agent Omega suite | 161 passed, 0 failed |
| Windows package | Version sync and setup smoke passed; .NET Release build passed with the existing `MSB3277` warning; the packaged engine hash matched the released Windows asset |

The native run passed every source and platform job. Its automated release
packaging step failed afterward because the checkout lacked the local annotated
tag object used by that validator. The release was therefore assembled from
the run's exact immutable artifacts, re-hashed against the published manifest,
privacy-scanned, and uploaded without rebuilding them. This limitation does
not weaken the native binary proof; it is retained here so the release record
does not misstate a red packaging job as green.

## Scope not verified

- No Linux desktop shell exists yet; Linux assets are the engine fork only.
- The macOS desktop package is Apple Silicon-only; the Intel engine is ready for
  adopters but does not create an Intel Agent Omega app.
- macOS signing/notarization was not performed for these unsigned engine assets.
- The optional paid cloud/API browser suite was not run as a release gate.
