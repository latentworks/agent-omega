# v2.7 engine migration

Agent Omega v2.7.0 requires the matching v2.7.0 Agent Omega engine fork. The
task-quality lifecycle is enforced by the engine; an upstream OpenCode binary
or an earlier Agent Omega engine is intentionally rejected at startup rather
than being treated as an unenforced prompt convention.

## Protocol 2 adopter note

Protocol 2 adds a clean-room review path without weakening the existing
healthy-subagent review (HSS) path.

- HSS still uses one structured terminal review result.
- When no healthy independent subagent is available, CRAP starts one isolated,
  context-free review call and returns one bounded plain-language adversarial
  report. The report is limited to 24 KiB of UTF-8 and SHA-256-bound to durable
  engine state.
- The engine reloads and verifies the stored report, then delivers it once as a
  tagged synthetic user turn. Replaying the same review identity is idempotent.
- A repaired-plan checkpoint is accepted only after that exact durable turn is
  delivered. Artifact closure also requires a new settled execution or
  verification receipt after delivery.
- Empty, oversized, malformed, mismatched, or partially delivered reports fail
  closed and remain recoverable.

Startup requires task-quality protocol 2 and all seven attested features:
`tool-admission`, `isolated-review`, `trusted-origin`, `lifecycle-cas`,
`plain-review-report`, `review-address-gate`, and `review-resume`.

## Windows upgrade

1. Update the repository to v2.7.0 and run `node setup.mjs`. Existing model,
   council, and memory data are preserved.
2. Download `opencode-windows-x64.exe` and `SHA256SUMS-omega-v2.7.0.txt` from
   the [engine v2.7.0 release](https://github.com/latentworks/opencode-omega/releases/tag/omega-v2.7.0).
3. Rename the binary to `opencode.exe` and put it at `engine\opencode.exe` in
   the repository root.
4. Verify the binary against the published checksum file and
   `SOURCE-PROVENANCE.json` before running it:

   ```powershell
   Get-FileHash .\engine\opencode.exe -Algorithm SHA256
   ```

5. Rebuild with `dotnet build -c Release` and launch the rebuilt application.

`node scripts\smoke.mjs` checks that the engine file is present, but only the
app startup handshake proves v2.7 compatibility.

## Source and platform scope

The matching public source is
[`latentworks/opencode-omega`](https://github.com/latentworks/opencode-omega),
tagged `omega-v2.7.0`. The engine release provides native assets, checksums,
and machine-readable source provenance. Agent Omega's desktop shell supports
Windows x64 and macOS Apple Silicon; Linux and macOS Intel engine assets are for
engine adopters and future shell work, not desktop-app packages.
